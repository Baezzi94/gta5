## 롤백 가이드

### 1) 코드 롤백 (Git)
- 특정 버전 상태 보기: `git checkout v0.4.0`
- 안전하게 되돌리기(이력 보존, 권장): `git revert <커밋해시>` 후 `git push`
- 강제로 그 시점으로(주의): `git reset --hard v0.4.0` (이미 push했다면 지양)
- main에 push하면 **Vercel이 자동 재배포**된다.

### 2) 배포만 빠르게 롤백 (Vercel)
- Vercel 대시보드 → 프로젝트 → **Deployments** → 이전 정상 배포 → **Promote to Production**
- 코드 수정 없이 즉시 이전 버전으로 서비스 전환 가능 (긴급 대응용)

### 3) ⚠️ DB(마이그레이션) 주의
- 마이그레이션은 **앞으로만(forward-only)** 적용했다. 코드를 옛 버전으로 되돌려도 **DB 스키마는 그대로**다.
- 보통 신버전 DB는 구버전 코드와 호환되지만(컬럼 추가 위주), 문제가 의심되면 롤백 전에 알려줄 것 → 그에 맞는 역방향 SQL을 안내한다.
- 데이터 초기화가 필요하면 `supabase/scripts/reset_testdata.sql` 사용(사장 계정만 보존).
