# 셋업 절차 (1회, 유저 콘솔 작업)

## 1. Supabase 프로젝트
1. supabase.com → **New project**: 이름 `blackout-intel` (⚠️ 토킹바 프로젝트와 별개!)
2. 리전 아무거나(서울 가능하면 서울), DB 비밀번호 저장해두기

## 2. 디스코드 OAuth 앱
1. discord.com/developers/applications → **New Application**: 이름 무관 (예: "BO")
2. OAuth2 탭 → Redirects에 추가:
   `https://<프로젝트ref>.supabase.co/auth/v1/callback`
   (프로젝트ref는 Supabase 대시보드 URL/Settings에서 확인)
3. **Client ID / Client Secret** 복사

## 3. Supabase에 디코 연결
1. Supabase → Authentication → Providers → **Discord**: 위 ID/Secret 입력, Enable
2. Authentication → URL Configuration:
   - Site URL: `http://localhost:5173` (배포 후 Vercel URL로 갱신)
   - Redirect URLs: `http://localhost:5173/**` (배포 후 `https://<vercel-domain>/**` 추가)

## 4. Storage 버킷
- Storage → **New bucket**: 이름 `tip-photos`, **Private** (Public 체크 해제)

## 5. 로컬 환경변수
`갱활동/app/.env.local` 파일 생성 (git에 안 올라감):
```
VITE_SUPABASE_URL=https://<프로젝트ref>.supabase.co
VITE_SUPABASE_ANON_KEY=<Settings → API → anon public key>
```

## 6. 마이그레이션 실행
- Supabase → SQL Editor → `supabase/migrations/0001_init.sql` 내용 붙여넣고 Run
- 이어서 `0002_rls.sql` Run
- Table Editor에서 테이블 7개 + categories 5행 확인

## 7. 첫 관리자 지정 (본인)
- 앱에서 디코 로그인 + 온보딩 1회 완료 후,
- Supabase → Table Editor → profiles → 내 행:
  `status = active`, `role = intel_chief` 로 직접 수정
- 이후 승인·계급 부여는 앱 [관리] 탭에서
