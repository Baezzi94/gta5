# Black Out 갱활동 앱 — 전체 로드맵 (MVP → 완성)

작성: 2026-07-13. 각 버전 착수 시점에 해당 버전의 상세 구현 플랜을 `plans/`에 새로 작성한다 (선행조건 미확정 상태에서 코드 레벨 플랜을 미리 짜지 않는다 — YAGNI).

원칙: 버전 하나 = 배포 가능한 완결 단위. 완료 시마다 태그(`갱활동-vX.Y.Z`).

## v0.1 — 정보부 시스템 MVP ★현재 진행

- 상세 플랜: `plans/2026-07-13-정보부시스템-MVP-플랜.md` (12태스크)
- 제보→검증→P0~P3 열람 + 인물 DB(전화번호 키) + 조직 관리 + 배포
- 선행조건: Supabase/디코 OAuth 셋업(T2, 유저), Vercel(T12, 유저)

## v0.2 — 사업 대시보드 (보스 1순위 요청)

- 형님 권한(P0/P1)에서 보이는 사업 보드: 사업별 카드(블랙모터스/세차장/하우스), 구상 메모, 진행상황(구상→건의→승인→운영), 수익구조 기록
- 데이터: `businesses`(이름, 상태, 수익구조 메모, clearance), `business_logs`(진행 기록)
- 선행조건: 없음 (구상 단계 그대로 담는 게 목적). v0.1 배포 직후 착수
- 보스가 직접 편집 가능하게 → "비즈니스 모델 같이 가꾼다" 약속 이행

## v0.3 — 디코 봇 "정보부장(앱)"

- 정식 디스코드 봇(APP 배지). 셀프봇 금지(약관)
- 기능: ① 진핑 전용 비밀 채널 입력 → 지정 채널 릴레이(페르소나 발화) ② 앱 이벤트 알림(새 제보 접수 → 정보부 채널, 채택 공지)
- 구현: Vercel 서버리스(Edge Function) + Discord REST, 또는 소형 상시 봇(무료 호스팅) — 착수 시 결정
- 선행조건: 디코 개발자 포털 봇 생성(유저), 갱단 서버 초대(대부 협조)

## v0.4 — 보고서

- 정보부장이 채택 정보 N건을 묶어 보고서 작성·발행(P등급 지정), 열람은 등급 규칙 동일
- 데이터: `reports`(제목, 본문, clearance, 발행일), `report_tips`(연결)
- 선행조건: v0.1. (v0.3 있으면 발행 시 봇 공지 연동)

## v0.5 — 사회관계도

- 인물 간 관계 그래프. 전화번호=노드 ID, 이름은 라벨(다중 이름 = 한 노드)
- 데이터: `person_relations`(person_a, person_b, 관계유형: 가족/조직동료/거래/적대/연인 등, 메모)
- UI: 그래프 뷰(캔버스/SVG 자체 구현 또는 경량 라이브러리 — 착수 시 결정), 인물 상세에서 관계 추가
- 선행조건: v0.1 인물 DB에 데이터 쌓여 있을수록 가치. tip_persons 공동 등장도 자동 엣지 후보

## v0.6 — 정보 거래

- 채택 정보에 판매 플래그: 보스가 판매 가능/불가 결정(전용 큐)
- 정보비 장부: 매입(정보비 지출) / 판매(수익) 기록, 잔액 집계
- 데이터: `tips.sale_status`(none/candidate/approved/denied/sold), `info_ledger`(구분, 금액, 상대, 메모, tip_id)
- 선행조건: v0.1. 보스 UI 필요(P0 전용 탭)

## v0.7 — 수금·재정 자동화

- **선행조건: 차미슬 인터뷰(15~30분) — 이거 없이 착수 금지**
- 예상 범위: 수금대장(대상, 주기, 금액, 납부 기록), 미납 추적, 자동 집계, (v0.3 있으면) 미납 알림 푸시
- 인터뷰에서 확정할 것: 수금 대상 단위(구역? 사업장?), 주기, 금액 산정 방식, 누가 걷는지, 현재 기록 방식

## v0.8 — 정산 계산기 (편의점털이 등)

- 라운드별 참가자 체크 → 정산 시점에 라운드×참가 횟수 기준 엔빵 자동 계산
- 모드: 균등 엔빵 / 몰아주기 / 사다리타기(랜덤)
- 데이터: `heists`(컨텐츠, 일시), `heist_rounds`, `heist_participants`, 정산 스냅샷
- 선행조건: 엔빵 규칙 상세 확정(라운드 정의, 수익 입력 주체) — 착수 시 진핑이 실플레이 기준으로 확정

## v0.9 — 사업 장부 (블랙모터스/세차장/하우스)

- 판매 기록 입력 → 세금 80% / 조직 20% / (판매원 커미션?) 자동 계산, 월별 집계 → v0.2 대시보드에 연결
- 세차용품: 품목/수량(1·5·10)/단가 장부 — 토킹바 술 판매 패턴 이식
- **선행조건: 운영진 답변 3건** (시스템 상점 여부 / 세금 자동공제 여부 / 재고 방식) + 하우스 스펙
- 사업회신_디스코드.txt의 질문들이 이 버전의 입력값

## v1.0 — 자산 통합 대시보드

- 조직 자산 현황 종합: 정보비 장부(v0.6) + 수금(v0.7) + 정산(v0.8) + 사업 장부(v0.9) 흐름을 한 화면에
- 보스 전용 뷰 + 기간별 추이
- 선행조건: v0.6~0.9 중 최소 2개 이상 운영 데이터 존재

## 순서 유연성

- v0.2와 v0.3은 v0.1 직후 병렬 가능 (서로 독립)
- v0.7~0.9는 외부 선행조건(인터뷰/운영진/서버 컨텐츠) 풀리는 순서대로 당겨 착수
- 보스 요청이 들어오면 해당 버전을 앞당김 — 스폰서 우선
