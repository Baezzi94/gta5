# 운영앱 V1-B (운영 화면) Implementation Plan

> **For agentic workers:** 인라인 실행(executing-plans). 순수 로직은 TDD, 데이터접근·UI는 빌드+수동 E2E로 검증.

**Goal:** 실제 운영에 쓰는 화면 구축 — 멤버/세션/출근부/손님/밴/예약. 완료 시 사장이 세션을 열고, 공주님 출근을 잡고, 손님을 등록/밴하고, 예약판에서 가용성 기반으로 예약을 잡을 수 있다.

**Architecture:** 기능별 데이터접근 모듈(`src/lib/<feature>.js`)이 supabase를 직접 호출(보안은 RLS). 페이지는 역할별 네비게이션(Layout)을 통해 접근. 순수 로직(추천인 검색, 예약→busy 변환)은 분리해 Vitest로 테스트.

**Tech Stack:** React 18, react-router-dom 6, @supabase/supabase-js 2, Vitest.

## Global Constraints

- 권한: 클라이언트 `can()`로 UI 게이팅 + RLS로 실제 보안(이중).
- 손님 phone = 실질 ID(unique). 익명 표시는 `dailyAnonCode(phone, new Date())`.
- 예약 슬롯은 분 단위(세션 오픈 기준). 가용성은 `src/lib/availability.js` 사용.
- 모든 금액·데이터는 인게임 RP 기준.

## File Structure

```
src/
├─ lib/
│  ├─ members.js / members.test.js     # 멤버 CRUD + 추천인 phone·닉 검색
│  ├─ sessions.js                       # 세션 CRUD + 현재 세션
│  ├─ attendance.js                     # 출근부 CRUD
│  ├─ customers.js                      # 손님 CRUD + phone 검색
│  ├─ bans.js                           # 밴 CRUD
│  ├─ reservations.js / reservations.test.js  # 예약 CRUD + reservationsToBusy
├─ components/
│  ├─ Layout.jsx                        # 역할별 사이드 네비 + 로그아웃
│  └─ Field.jsx                         # 간단 입력 래퍼(선택)
├─ pages/
│  ├─ Members.jsx                       # 멤버 관리(owner)
│  ├─ Sessions.jsx                      # 세션 관리(owner)
│  ├─ Attendance.jsx                    # 출근부
│  ├─ Customers.jsx                     # 손님 관리
│  ├─ Bans.jsx                          # 밴 관리
│  └─ Reservations.jsx                  # 예약판
└─ app/App.jsx                          # 라우트 확장(중첩)
```

### Task 1: 레이아웃 + 역할별 네비 + 라우트 확장
- `src/components/Layout.jsx`: 좌측 메뉴(역할별 `can()`로 필터), 상단 역할 표시, 로그아웃. `<Outlet/>` 렌더.
- `src/app/App.jsx`: `/` 아래 중첩 라우트 — members, sessions, attendance, customers, bans, reservations. 각 ProtectedRoute.
- 검증: 빌드 + 로그인 후 메뉴 노출.

### Task 2: 멤버 모듈 + 추천인 검색 (TDD)
- `src/lib/members.test.js`: `normalizePhone()`, `searchMembers(members, query)` (phone 또는 닉 부분일치).
- `src/lib/members.js`:
  - `normalizePhone(s)` — 숫자만 남김
  - `searchMembers(members, query)` — query를 normalize/소문자화해 phone 또는 name 부분일치 필터
  - `listMembers()`, `createMember({name, phone, type, referred_by, profile_photo_url, memo})`, `updateMember(id, patch)`, `deactivateMember(id)` (supabase)
- `src/pages/Members.jsx`: 목록 + 추가 폼(추천인은 검색해서 선택). owner 전용.
- 검증: TDD 통과 + 빌드 + 멤버 추가/조회.

### Task 3: 세션 모듈
- `src/lib/sessions.js`: `listSessions()`, `createSession(date)`, `openSession(id)`, `closeSession(id)`, `getOpenSession()`.
- `src/pages/Sessions.jsx`: 세션 목록, 새 세션 생성, 열기/마감 버튼(owner).
- 검증: 빌드 + 세션 생성·열기.

### Task 4: 출근부(attendance)
- `src/lib/attendance.js`: `listBySession(sessionId)`, `addPlanned(sessionId, memberId)`, `checkIn(id)`, `checkOut(id)`, `setAvailableSlots(id, n)`, `removePlanned(id)`.
- `src/pages/Attendance.jsx`: 현재(열린) 세션의 공주님 출근 목록. 사장/스탭: 예정 추가·가용 슬롯 입력. 공주: 본인 출근 체크.
- 검증: 빌드 + 출근 추가·체크·슬롯 입력.

### Task 5: 손님(customers)
- `src/lib/customers.js`: `searchByPhone(phone)`, `listRecent()`, `createCustomer({phone, nickname, memo, birthday, preferred_princess, referred_by})`, `updateCustomer(id, patch)`.
- `src/pages/Customers.jsx`: phone 검색/등록, 마케팅 필드, 익명 표시(`dailyAnonCode`) 토글, 추천인 검색 선택.
- 검증: 빌드 + 손님 등록/검색, 익명코드 표시.

### Task 6: 밴(bans)
- `src/lib/bans.js`: `listBans()`, `createBan({customer_id, phone, reason})`, `liftBan(id)`, `isBanned(phone)`.
- `src/pages/Bans.jsx`: 밴 목록, 등록(사유), 해제. 손님 등록/예약 시 `isBanned` 경고.
- 검증: 빌드 + 밴 등록·해제.

### Task 7: 예약판(reservations) + reservationsToBusy (TDD)
- `src/lib/reservations.test.js`: `reservationsToBusy(reservations)` → `[{start,end}]` (취소·노쇼 제외).
- `src/lib/reservations.js`:
  - `reservationsToBusy(reservations)` (순수)
  - `listBySession(sessionId)`, `createReservation({session_id, customer_id, princess_id, start_min, end_min})` — 생성 전 `isAvailable` 검사, 불가 시 throw
  - `setStatus(id, status)`
- `src/pages/Reservations.jsx`: 열린 세션의 공주님별 예약 타임라인, 예약 추가(가용성 체크·연장 알럿), 상태 변경(진행/완료/노쇼/취소).
- 검증: TDD 통과 + 빌드 + 예약 생성/충돌 차단.
