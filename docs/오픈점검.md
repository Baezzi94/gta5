# 🚀 공주님 클럽 오픈 전 점검 체크리스트

> 실제 오픈(손님·공주 받기) 직전에 이 순서대로 1회 훑는다. 게임 내 RP 운영 기준.

---

## 1. 데이터 초기화 (테스트 흔적 지우기)
- [ ] Supabase → SQL Editor에서 **`supabase/scripts/reset_testdata.sql`** 실행
  - 사장(qodiaos@gmail.com) 계정/멤버만 남기고 거래·예약·출근·정산·밴·손님·기타 계정 전부 삭제
  - 메뉴판(양주/맥주/담배 등 카탈로그)은 **보존**됨
  - 실행 후 하단 확인 쿼리로 members=1(사장), 나머지 0 확인
- [ ] 운영앱 로그인 → 각 탭이 깨끗한지 눈으로 확인 (예약판/수금정산/손님/밴)

## 2. 가입코드 확인 & 배포
- [ ] 운영앱 **멤버 탭** 상단에서 역할별 현재 코드 확인 (사장만 보임)
  - 사장 / 스탭 / 삐끼 / 공주 4종
- [ ] 테스트 중 노출됐던 코드는 **재발급** 버튼으로 새로 뽑기
- [ ] 각 사람에게 **본인 역할 코드만** 전달 (`가입코드.txt` 템플릿 참고)
  - 공주님에게는 "추천인(영입한 사람) 전화/닉" 입력하라고 안내 → 영입 인센티브 자동 매핑
- [ ] ⚠️ `가입코드.txt`는 절대 커밋 금지 (gitignore 확인)

## 3. Supabase 설정 점검
- [ ] Auth → **이메일 인증(Confirm email) OFF** 인지 확인
  - 켜져 있으면 "email rate limit exceeded" / 가입 후 로그인 안 됨
- [ ] 앱은 **publishable(anon) 키만** 사용 (secret 키 노출 금지 — 이전에 노출됐으면 회전 완료됐는지 확인)
- [ ] Vercel 배포 최신인지 (`https://gta5-rho.vercel.app` 접속 → 최근 변경 반영 확인)

## 4. 메뉴판 확인 (수금/정산 정확도)
- [ ] 메뉴 탭에서 판매가/도매가 최신인지: 양주 30 / 프리미엄 50 / 맥주 3 / 담배 2 (만)
- [ ] 기본요금 TC 5만(맥주 2잔 포함) / 대화료 25만 — 포스터와 일치

## 5. 전체 플로우 리허설 (오픈 전 1바퀴)
> 실제 코드로 더미 공주 1명·손님 1명 만들어 한 바퀴 돌려보고, 끝나면 지운다.
- [ ] 공주 코드로 가입 → 로그인됨 / 역할 = 공주 확인
- [ ] 공주 **출근(체크인)** → 예약탭에서 그 공주로 **예약 생성 가능**한지
- [ ] 출근 안 한 공주는 **예약이 막히는지** (게이팅 확인)
- [ ] 예약 **진행** → TC·대화료 charge 자동 생성 확인
- [ ] 주류/메뉴 판매 폼으로 양주 1병 추가 → 수금정산에 반영
- [ ] **정산 분배** 표가 출근 중인 사람에게만 배분되는지 (사장 미출근 시 사장 빠지는지)
- [ ] 사장 계정으로 **지급/미지급 토글**, 삭제 버튼이 **사장만** 보이는지
- [ ] 손님 **밴** 등록 → 그 손님 재예약/거래가 **차단**되는지
- [ ] 리허설 끝 → 더미 멤버/손님 삭제 (FK 에러 없이 지워지는지)
